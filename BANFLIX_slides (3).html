05 / 08
Data Flow

데이터 플로우

1
사용자 입력 수집
장르 칩(복수선택), 기분 칩(단일선택), 러닝타임 칩(단일선택) 또는 검색어를 입력받습니다.
chip.addEventListener('click') / input.oninput
2
필터 로직 실행
MOVIES 배열을 genres·moods·runtime 세 축으로 교차 필터링합니다. 검색 시에는 KOBIS·Netflix 데이터를 합산해 전체를 대상으로 소문자 포함 검색을 수행합니다.
MOVIES.filter(m => genreOk && moodOk && timeOk)
3
결과 정렬 및 슬라이싱
추천 모드에서는 Math.random() 셔플 후 최대 4편으로 제한합니다. 검색 모드에서는 매칭된 전체를 순서대로 반환합니다.
matched.sort(() => Math.random() - 0.5).slice(0, 4)
4
카드 렌더링
makeCard() 함수가 장르 썸네일·플랫폼 배지·검색 하이라이트를 포함한 HTML 문자열을 생성해 innerHTML로 주입합니다. CSS fadeUp 애니메이션이 순차 적용됩니다.
el.innerHTML = matched.map(makeCard).join('')
5
상세 모달 열기
카드 data-movie 속성에 JSON 인코딩된 객체를 저장해 클릭 이벤트 위임으로 openDetail()을 호출합니다. body 스크롤을 잠그고 바텀시트를 슬라이드업합니다.
JSON.parse(decodeURIComponent(card.dataset.movie))

Architecture Decision Records

아키텍처 결정 기록

ADR-001
단일 HTML 파일 구조 채택
ACCEPTED ▶
Context

별도 빌드 도구 없이 팀원 전원이 즉시 실행·수정 가능한 배포 환경이 필요했다. GitHub Pages 정적 호스팅을 목표로 했다.

Options Considered
✓ 단일 HTML (Vanilla JS) React + Vite Next.js SSG
Decision

HTML·CSS·JS를 한 파일에 번들. 빌드 파이프라인 없이 브라우저에서 바로 실행 가능하며 GitHub Pages 배포가 최단 경로.

Consequences
✅ 배포 즉시성 — 파일 업로드만으로 라이브
✅ 의존성 제로
⚠️ 파일 비대화 시 유지보수 어려움
ADR-002
기분(Mood) 기반 필터 축 도입
ACCEPTED ▶
Context

사용자가 장르보다 감정 상태로 영화를 고르는 경우가 많다는 UX 리서치 결과를 반영해야 했다.

Options Considered
장르 단독 필터 ✓ 장르 + 기분 + 러닝타임 3축 필터 AI 추천 알고리즘
Decision

기분(설레는·무서운·웃긴·울고 싶은 등) 태그를 영화 데이터에 메타로 추가하고, 장르·러닝타임과 AND 조합으로 필터링.

Consequences
✅ 직관적 탐색 UX
✅ 서버리스 구현 가능
⚠️ 큐레이션 영화 수 확장 시 태깅 비용 증가
ADR-003
영화 데이터 하드코딩 (인라인 JSON)
ACCEPTED ▶
Context

외부 API 키 없이 오프라인에서도 동작해야 하고, MVP 단계에서 데이터 규모가 20편 수준으로 제한적이다.

Options Considered
TMDB API 실시간 연동 별도 JSON 파일 fetch ✓ HTML 내 인라인 JS 배열
Decision

영화 20편을 JS 배열로 HTML에 내장. API 키·CORS·네트워크 의존성 없이 완전 정적 동작 보장.

Consequences
✅ 오프라인 동작
✅ API 비용 0원
⚠️ 데이터 갱신 시 파일 직접 수정 필요
ADR-004
CSS 변수 기반 넷플릭스 다크 테마 시스템
ACCEPTED ▶
Context

OTT 서비스 분위기를 살리면서 다크모드 일관성 유지가 필요했고, 컴포넌트 간 색상 동기화가 요구됐다.

Options Considered
Tailwind CSS 인라인 스타일 직접 기입 ✓ CSS :root 변수 토큰 시스템
Decision

:root에 --red, --gold, --bg, --surface 등 디자인 토큰을 정의하고 전체 컴포넌트가 변수로 참조. 빌드 없이 테마 일관성 확보.

Consequences
✅ 색상 변경 1곳에서 전체 반영
✅ 라이트/다크 테마 전환 용이
⚠️ IE11 미지원 (프로젝트 범위 외)
